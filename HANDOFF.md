# Purinton Analytics (pa-expert.com) — Handoff

A fast, fully static, SEO- and GEO-optimized marketing site for **Purinton
Analytics, LLC** — forensic vocational expert & life care planning practice
(principal: Jason C. Purinton, LPC, CRC, CVE, FVE, ABVE/F, IPEC).

- **Branch:** `claude/pa-expert-website-redesign-8wb1bh` (merged into `main`)
- **Pages:** 343 total — home, about, 5 service pages, practice-areas hub + 4
  areas, contact, privacy, 404, **locations hub + 7 state pages + 327 city pages**
- **Stack:** Hand-rolled **static HTML/CSS/JS**, generated by one Python script
  (`build.py`). No framework, no Node, no build step required to *host*.

---

## 1. Quick start

```bash
# Regenerate every page (after editing build.py / content / data)
python3 build.py

# Preview locally
python3 -m http.server 8000
#  → http://localhost:8000/
```

`build.py` writes all `index.html` files in place plus `sitemap.xml`,
`robots.txt`, `llms.txt`, `site.webmanifest`, and `CNAME`. Commit the generated
output — hosting needs no build step.

## 2. Architecture

Everything is driven from **`build.py`** so 343 pages stay consistent (shared
`<head>`, header, footer, SEO tags, JSON-LD). You edit data/templates there and
re-run; you do **not** hand-edit the generated `index.html` files.

Key symbols in `build.py`:

| What | Where |
|---|---|
| Global business facts (name, phone, email, address, theme color, geo) | `SITE` dict (top) |
| Top nav | `NAV` |
| Services (cards + detail pages) | `SERVICES`, `DETAILS` |
| Practice areas | `PRACTICE_AREAS`, `DETAILS` |
| **Locations data** (states, cities, courts, comp authority, industries) | `STATES`, `STATE_INDUSTRIES` |
| City-page copy variants (anti-duplication) | `CITY_INTRO`, `CITY_LOCAL`, `CITY_ECON`, `CITY_TRUST`, `city_faq_pool()` |
| `<head>` / meta / OG / geo | `head()` |
| Header + logo | `header()` |
| Footer | `footer()` |
| Schema fragments (JSON-LD) | `org_schema()`, `person_schema()`, `service_schema()`, plus per-page schema |
| Page registry (what gets built) | `build_pages()` |
| Sitemap / robots / llms.txt / manifest | `write_meta_files()` |

Styling: `assets/css/style.css` (design system in `:root`, components,
animations). Behavior: `assets/js/main.js` (mobile nav, header shadow,
IntersectionObserver scroll-reveal).

### Repo layout
```
build.py                      generator (source of truth)
index.html, 404.html          generated
about/ services/ ...          generated clean-URL pages
locations/<state>/<city>/     generated location pages
assets/css/style.css

```

### 🔴 Do before launch
1. **Open Graph image → PNG.** `assets/img/og-default.*` is currently SVG, which
   Facebook/LinkedIn/X **do not render** in link previews. Export a **1200×630
   PNG**, save as `assets/img/og-default.png`, then in `build.py` change the
   `og_img` line in `head()` (and the `"image"` in `org_schema()`) from `.svg`
   to `.png`, and `python3 build.py`. *(No image tooling existed in the build
   environment, so this is left as a manual step.)*
2. **Contact form endpoint.** `contact_body()` posts to a Formspree placeholder
   `https://formspree.io/f/your-form-id`. Create a Formspree (or Netlify Forms /
   other) form and replace the ID. Test a real submission.
3. **Confirm email.** `SITE["email"]` defaults to `info@pa-expert.com` — verify
   or change.

### 🟡 Branding (you chose "match old pa-expert.com")
The site is themed with CSS variables, so re-skinning is fast once assets land.
- **Logo:** drop `assets/img/brand/logo.svg`, then in `header()` swap the
  `.brand-mark` "PA" block for `<img src="/assets/img/brand/logo.svg" …>`.
- **Favicon:** replace `favicon.svg` (or add `favicon.png` and update the
  `<link rel="icon">` in `head()`).
- **Colors:** set the real hex values in `:root` of `style.css` (`--navy-*`,
  `--gold-*`, etc.) and `SITE["theme"]` in `build.py`. (I can extract the exact

- **Site photography / backgrounds:** drop into `assets/img/source/` (via
  GitHub web upload or git — paste won't work for binary), then they get wired

## 6. Deployment
- Static — host the repo root anywhere (GitHub Pages, Netlify, Cloudflare Pages,
  S3). `CNAME` is set to `pa-expert.com` for GitHub Pages custom domain.
- **GitHub Pages:** Settings → Pages → deploy from branch `main` (root). DNS:
  point `pa-expert.com` at GitHub Pages per their docs.
- **Note on default branch:** the repo's default branch was the redesign branch.
  `main` now contains all work; you may want to set `main` as the default in
  GitHub repo settings.

## 7. Continuing in a local session
1. `git fetch && git checkout main` (or the redesign branch).
2. Edit content/data in `build.py`, styles in `assets/css/style.css`.
3. `python3 build.py` → `python3 -m http.server 8000` to preview.
4. To add a city/state: edit `STATES` (and `STATE_INDUSTRIES`) and rebuild.
5. Commit the regenerated output.

Add real assets to `assets/img/brand/` and `assets/img/source/` and follow §3
to wire them in. The whole theme keys off CSS variables, so brand matching is a
small, contained change.
