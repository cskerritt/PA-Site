## 6. Architecture & Build Approach

- **Single source of truth stays `build.py`.** Extend data structures
  (`SITE`, `PRACTICE_AREAS`, `DETAILS`, plus new `OFFICES`, `ARTICLES`,
  `AFFILIATIONS`) rather than hand-editing generated HTML.
- **Refactor watch:** `build.py` is already large. As we add pages, keep page-type
  builders as focused functions and data as data; if a section grows unwieldy,
  split into a small module (e.g. `content.py` for data, `build.py` for rendering)
  — only if it genuinely improves clarity, no gratuitous refactor.
- **Asset pipeline:** copy brand assets into `assets/img/brand/`; reference WebP
  with appropriate `width`/`height`/`alt`/`loading="lazy"`. Generate favicon +
  OG PNG (1200×630) into `assets/img/`.
- **Regeneration:** `python3 build.py` rebuilds all HTML + `sitemap.xml`,
  `robots.txt`, `llms.txt`, `site.webmanifest`, `CNAME`. `llms.txt` and sitemap
  must reflect every new page.
- **SEO/GEO consistency:** every new page gets title/description/canonical/OG +
  JSON-LD; AI-crawler allowances in `robots.txt` preserved.
